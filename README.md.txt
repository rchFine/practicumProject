Репозитории Git – контейнеры для отслеживаемых проектов.
Их можно разделитить на два основных типа:

Локальный репозиторий хранится на вашем компьютере, где вы можете работать над локальной версией проекта.
Удаленный репозиторий хранится вне вашей локальной системы, обычно на сервере. Это позволяет разработчикам делиться кодом.

Чтобы создать новый репозиторий и начать отслеживать проект с помощью Git, в терминале откройте главную папку проекта и наберите следующую команду:
Подготовка к коммиту
Перед совершением коммита файлы необходимо подготовить, т. е. поместить их в некую область хранения. Для этого используется команда (git add).

Коммит
Коммит – это снимок кода в определенный момент времени, который сохраняется в истории коммитов репозитория. После добавления всех отслеживаемых файлов с помощью команды git add, можно совершить коммит: git commit -m "Сообщение коммита"

Чтобы увидеть все коммиты проекта, используйте следующую команду: git log
